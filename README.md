# codyssey-week3
# Mini NPU Simulator

## 실행 방법

```bash
python main.py
```

- 1번: 사용자 입력(3x3) 모드
  - 필터 A, B, 패턴을 각각 3줄씩 입력
  - 형식 오류 발생 시 입력을 다시 받음
- 2번: data.json 분석 모드
  - `data.json`를 현재 폴더에서 읽어 처리
  - 실패 사례는 종료 없이 모두 출력하고 최종 요약
- 3번: 패턴 생성기
  - 원하는 N 입력 시 N×N Cross, X 패턴을 생성해 출력

## 구현 요약

- Python 3.8 이상에서 동작
- 외부 라이브러리 미사용, `json`, `time`, `re`만 사용
- 핵심 연산은 이중 반복문 기반 `MAC` 직접 구현
  - `MAC = sum(패턴[i][j] * 필터[i][j])`
- 라벨 정규화
  - `+`, `cross` → `Cross`
  - `x`, `X` → `X`
- 동점 판정 정책
  - `abs(score_a - score_b) < 1e-9` → `UNDECIDED`
- PASS/FAIL 판단
  - 정규화된 라벨(standard label) 기준으로 비교
- JSON 스키마 실패(크기 불일치, 파싱 실패 등)는 예외로 터뜨리지 않고 케이스 단위로 기록

## 결과 리포트(README에 제출할 항목)

### 실패 원인 분석

1. 입력/스키마 오류로 인한 실패는 케이스 단위로 분리해 기록한다.  
2. 가장 흔한 실패는 패턴 크기와 대응 필터 크기 불일치이다.  
3. expected 라벨 표기(`+`, `x`)가 그대로 섞이면 비교가 깨지므로 정규화가 필요하다.  
4. Cross/X 문자열(대소문자/기호 차이) 처리 없이 비교하면 오탐이 발생한다.  
5. 부동소수점 연산 시 `1e-12` 정도의 미세한 오차가 축적되어 점수가 완전히 같아도 미세차이로 보일 수 있다.  
6. epsilon 비교를 넣으면 동일한 수치 오차를 하나의 동점으로 통일하여 판정을 안정화한다.  
7. 동점(UNDECIDED)은 기대 라벨과 다르게 처리해 PASS로 오판될 수 있으므로 별도 상태로 분기한다.  
8. 필터/패턴 파싱 실패를 예외로 중단하면 전체 테스트가 멈추므로, 본 과제는 실패를 누적 저장한다.  
9. 출력 라벨은 표준 라벨을 기준으로 기록해 케이스 추적을 일관되게 한다.  
10. 실패가 0개인 경우에도 “왜 0개인지”를 문서화하면 재현성 검증에 유리하다.

### 시간 복잡도 분석

- 한 번의 MAC 연산은 각 원소를 1회 방문하므로 `N×N`회 곱셈/덧셈이 필요.
- 따라서 시간 복잡도는 `O(N^2)`이다.
- 반복 측정 평균은 I/O 제외 구간에서 수행되어 순수 연산만 비교 가능하다.
- `N`이 커질수록 수행 시간이 제곱적으로 증가하므로, NPU 관점에서는 병렬처리와 캐시 지역성이 성능을 좌우한다.
- 고정 반복 횟수(10회)로 평준화하면 크기별 추세 비교가 가능하다.

### 보너스 항목: 1차원 배열 최적화 비교

- 2차원 배열은 `matrix[i][j]` 이중 인덱싱으로 합성곱을 수행.
- 1차원 배열은 `flatten` 후 `idx` 단일 인덱스로 같은 곱셈·누산을 수행.
- 성능 표는 `2D 평균(ms)`와 `1D 평균(ms)`를 함께 출력해 동일 입력에서 반복 횟수(10회) 기반으로 비교.
- 동일 연산량(`N²`)에서도 접근 패턴(메모리 캐시) 차이로 미세한 성능 차이가 발생할 수 있음을 확인.
