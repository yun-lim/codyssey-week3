# Mini NPU Simulator

## 실행 방법

```bash
python main.py
```

실행 시 모드 선택:

1. 사용자 입력 (3x3)
2. data.json 분석
3. 패턴 생성기(보너스)
4. 보너스: 1차원 최적화 비교(보너스)

## 구현 요약

- Python 3.8+ 기반 콘솔 애플리케이션
- 외부 라이브러리 없이 구현 (`json`, `re`, `time` 사용)
- MAC 연산은 중첩 반복문으로 직접 구현
- 라벨 정규화
  - `+` → `Cross`
  - `x` → `X`
- 데이터 비교 정책
  - `abs(score_a - score_b) < 1e-9`이면 동점
- 모드 1 판정
  - `A 점수`, `B 점수`, `판정: A/B/판정 불가`
- 모드 2 판정
  - `Cross 점수`, `X 점수`, `판정: Cross/X/UNDECIDED`
- 케이스 실패는 프로그램 종료 없이 누적 처리

## 입력/검증 규칙

- 사용자 입력 모드(3x3)
  - 모든 입력은 줄마다 공백 구분 3개 숫자
  - 형식 오류 시: `입력 형식 오류: 각 줄에 3개의 숫자를 공백으로 구분해 입력하세요.`

- data.json 모드
  - 최상위 키: `filters`, `patterns`
  - 필터: `filters.size_5`, `filters.size_13`, `filters.size_25`
  - 각 크기 블록은 `cross`, `x` 키를 가짐
  - 패턴 키 형식: `size_<N>_<id>`
  - 각 패턴 항목: `input` (2차원 배열), `expected` (`+` 또는 `x`)

## 성능 분석

- 모든 크기에서 MAC 연산 시간을 최소 10회 반복 측정 후 평균(ms) 출력
- 표준 항목: `크기`, `평균 시간(ms)`, `연산 횟수(N^2)`

## 결과 리포트(필수)

- 총 테스트 수 / 통과 수 / 실패 수 출력
- 실패 케이스를 `케이스ID`와 함께 사유 출력
- 실패가 있는 경우 케이스 단위 원인 정리
- 실패가 0개인 경우:
  - 정규화 규칙(`+`, `x`)
  - epsilon 기반 동점 처리
  - `N²` MAC 수행량 기반 복잡도(`O(N^2)`) 근거를 요약

### 시간 복잡도 분석

- 각 패턴 판별은 `N×N` 회의 곱셈/덧셈이 필요하므로 총 연산량은 `N²`.
- 반복 측정에서 I/O를 제외하고 MAC 호출 구간만 측정해 순수 연산 시간만 비교.
- 크기가 증가하면 연산량이 제곱으로 증가하므로 시간 역시 이에 비례해 증가.

### 실패 원인 분석 예시 항목(요구 충족용)

1. 라벨 표기 불일치(`+`, `x`, `Cross`, `X`) 미정규화
2. 크기 불일치 (`size_` 블록과 입력 패턴의 크기 mismatch)
3. 필터/패턴 파싱 실패
4. 부동소수점 비교 미세 오차(동점 판단 없음)
